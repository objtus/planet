# ダッシュボード UI仕様

## デザイン方針
- ダークテーマ対応（CSS変数で自動切替）
- シンプル・クール・情報密度高め
- 装飾最小限、フラットデザイン
- モバイル（iPhone）対応レスポンシブ
- フォント: システムデフォルト（var(--font-sans)）

---

## カレンダー画面（メイン）

### ナビゲーション
- `«` / `»` ボタン: 年単位で移動（**カレンダー表示のみ**が変わる。タイムライン見出し `#detail-title` は、実際に読み込んだ日/週/月/年のまま）
- `‹` / `›` ボタン: 月単位で移動（同上）
- 月・年のプルダウン: 直接選択（月は1〜12月、年は2017〜現在+1年）
- 「今日」ボタン: 当日に戻る
- カレンダー右上: 現在表示中の「○月」「○年」ボタン → それぞれ月ビュー・年ビューのタイムラインへ（カレンダー上の月/年は維持。view-tabs のモード切替とは別処理）
- 日付セルをクリック: その日を選択

### カレンダーグリッド
- 月曜始まり（ISO週）
- 左端列に週番号をピル型ボタン（W○○）で表示
  - スタイル: 角丸ピル、背景color-background-secondary、ボーダーあり
  - ホバー: 青くハイライト（color-background-info）
  - クリックで週ビューに切り替え
- 日付セルのヒートマップ: 凡例付近の `<select>` で指標を切替（投稿数 / 再生曲数 / 歩数 / 最高気温）。月・指標のたびに **`GET /api/heatmap?year=&month=&metric=posts|plays|steps|weather`** で取得し、**その月の min–max** を5段に線形正規化。指標ごとに `--heat-1`〜`--heat-5` の色相が異なる。投稿数・再生曲数は **0 の日は無着色**。
- 当月以外の日付（前月・翌月で週を埋めるセル）: 薄い文字色。クリックでその日へ移動
- 今日: 青いボーダー枠
- 選択中の日: color-background-info で塗りつぶし
- 凡例: セレクトの右にスウォッチ列と軸ラベル（例: 少→多、天気は低→高（℃））

## 詳細パネル（カレンダー下部）

### 表示モード切替タブ
日 / 週（W○○）/ 月 / 年 の4モード

- パネル上部の日付見出し（`#detail-title`）は、**現在表示しているタイムラインの期間**に対応する文字列。カレンダーの `«‹›»` で月だけ変えても、タイムラインを再読み込みしない限り見出しは変わらない（`calendar.js` の `tlTitle`）。

| モード | 表示内容 |
|---|---|
| 日 | 統計カード4枚 + 全ソースのタイムライン |
| 週 | 週次サマリー + 統計 + 全投稿一覧 |
| 月 | 月次サマリー + 統計 + 投稿一覧（遅延読み込み）|
| 年 | 年間統計 + 月別サマリー一覧 |

- サマリー本文（`/summaries` とカレンダー詳細のサマリーパネル）は **Markdown** を `marked` で HTML 化し、`DOMPurify` でサニタイズして表示する。年ビューの月一覧の抜粋はプレーンテキスト用に記号を簡易除去する。
- **週・月ビュー**のサマリーパネル見出し横に「この期間で生成」ボタンがあり、`POST /api/summaries/generate`（`summarizer.generate` を subprocess 実行、最大 900 秒）で再生成できる。ログ 0 件のときは API が `skipped: true` を返し、フロントで案内する。

### 統計カード（日ビュー）
4カラムグリッドで以下を表示:
- 投稿数（ソース別内訳をサブテキストで）
- 再生曲数（Last.fm）
- 歩数（Apple Watch）
- 天気（気温・天気概況・都市名）

### ソースフィルター
- 「すべて」ボタン: 全ソースをオンに戻す
- ソースタグ（ピル型トグルボタン）: クリックでオン/オフ
- オン時: ソース種別に応じた背景色・文字色
- オフ時: 無色・グレー文字
- 全オフ時: 「表示できる項目がありません」を表示

**ソースタグの色分け**
| 種別 | 背景 | 文字 | ボーダー |
|---|---|---|---|
| Misskey | #E1F5EE | #0F6E56 | #5DCAA5 |
| Mastodon | #EEEDFE | #3C3489 | #AFA9EC |
| Last.fm | #FCEBEB | #A32D2D | #F09595 |
| ヘルス | #FAEEDA | #854F0B | #EF9F27 |
| RSS | #EAF3DE | #3B6D11 | #97C459 |

### タイムライン
- 画像サムネイル: クリックでライトボックス拡大（閉じる・前後画像ナビ）
- ソースバッジ + 投稿内容 + **メタ行**（時刻・Last.fm 時は右端に削除）の構成（`.tl-meta-row` / `.tl-meta-row-left` / `.tl-meta-row-right`）
- **時刻リンク（カレンダー上のタイムライン）**: `logs.url` があるエントリは、時刻テキストが**ソース先（外部）**への `<a>`。URL がない行は `<span>` のみ（再リンクなし）。外部用の ↗ などは付けない。
- **Last.fm 行の「削除」**: 行右のボタン。確認後 `POST /api/logs/<id>/soft-delete` で `logs.is_deleted` を立てる（Last.fm のみ API が受理）。成功後は当該タイムラインを再取得し、ヒートマップも更新（`calendar.js`）。
- バッジ: favicon（16x16px の `<img>`）+ コンパクトなサーバー名
  - 実装時: `https://<instance>/favicon.ico` を `<img>` で表示
  - フォールバック: ソース種別の絵文字（🍣 Misskey / 🐘 Mastodon / 🎵 Last.fm / 🍎 ヘルス / 🌐 RSS）
- ブースト/リノートは通常投稿と区別して表示
  - `BT: <元テキスト>` / `RN: <元テキスト>` のプレフィックス
  - 文字色: color-text-secondary（やや薄く）
- ホバー: 背景をcolor-background-secondaryに

**サーバー名の短縮ルール**

## 検索画面（`/search`）

- **入力**: キーワード（`logs.content` の `LIKE '%…%'`）＋ソース（`source_id`）＋日付範囲（`date_from` / `date_to`）。`is_deleted = FALSE` のみ。最大 **200 件**（`dashboard/app.py` の `search()`）。
- **表示**: カレンダー日ビューに近いタイムライン風リスト（`templates/search.html`）。バッジの種別・絵文字は **`TYPE_INFO`**（`app.py`）とカレンダー側と揃える。
- **日時リンク**: クリックで **カレンダーの日ビュー**へ遷移（`/?view=day&date=YYYY-MM-DD`。日付は JST 表示に対応する `date_jst`）。検索では「いつか」をカレンダー上で位置づけやすくするため、カレンダー本体のタイムラインとは役割を分ける（カレンダー側の時刻はソース先リンク）。
- **Last.fm**: 行右に **「削除」**。確認ダイアログ（検索文脈向け文言）のあと、同じ **`POST /api/logs/<id>/soft-delete`** を `fetch` で呼び出し、成功時は **`location.reload()`** で結果を更新。
- 検索結果からソース URL への直接リンクや ↗ は**置かない**。

## ソース管理画面（`/sources`）

- 有効/無効トグル（従来どおり）
- 表示順: 行ごとの ▲/▼ → `POST /sources/<id>/move`
- 略称: インライン入力 → 確定で `POST /sources/<id>/rename`（DB の `short_name`。未設定時は自動短縮名）
- 収集: 行ごとの実行ボタン（種別にスクリプトがあるもの）と「全ソース今すぐ収集」。`POST /api/collect/<stype>`（`stype` は `weather` 等または `all`）。ログは画面のパネルに表示

## 技術スタック（ダッシュボード）

- バックエンド: Python / Flask
- テンプレート: Jinja2
- フロントエンド: Vanilla JS（フレームワークなし）
- CSS: CSS変数ベース（ライト/ダーク自動対応）
- グラフ: Chart.js（`static/js/chart.umd.min.js` ローカル配信。統計ページは月別・年別の切替あり）
- アクセス: Tailscale経由（http://<tailscale-ip>:5000）

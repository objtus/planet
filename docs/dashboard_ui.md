# ダッシュボード UI仕様

## デザイン方針
- ダークテーマ対応（CSS変数で自動切替）
- シンプル・クール・情報密度高め
- 装飾最小限、フラットデザイン
- モバイル（iPhone）対応レスポンシブ
- フォント: システムデフォルト（var(--font-sans)）

---

## カレンダー画面（メイン）

### ナビゲーション
- `«` / `»` ボタン: 年単位で移動
- `‹` / `›` ボタン: 月単位で移動
- 月・年のプルダウン: 直接選択（月は1〜12月、年は2017〜現在+1年）
- 「今日」ボタン: 当日に戻る
- 日付セルをクリック: その日を選択

### カレンダーグリッド
- 月曜始まり（ISO週）
- 左端列に週番号をピル型ボタン（W○○）で表示
  - スタイル: 角丸ピル、背景color-background-secondary、ボーダーあり
  - ホバー: 青くハイライト（color-background-info）
  - クリックで週ビューに切り替え（将来実装）
- 日付セルのヒートマップ: 青の濃淡5段階でその日の投稿数を表現
  - 0件: 無色
  - 1〜5件以上: #E6F1FB → #B5D4F4 → #85B7EB → #378ADD → #185FA5
- 今日: 青いボーダー枠
- 選択中の日: color-background-info で塗りつぶし
- 凡例: カレンダー下部左寄せに「投稿数 少→多」のカラースケール

## 詳細パネル（カレンダー下部）

### 表示モード切替タブ
日 / 週（W○○）/ 月 / 年 の4モード

| モード | 表示内容 |
|---|---|
| 日 | 統計カード4枚 + 全ソースのタイムライン |
| 週 | 週次サマリー + 統計 + 全投稿一覧 |
| 月 | 月次サマリー + 統計 + 投稿一覧（遅延読み込み）|
| 年 | 年間統計 + 月別サマリー一覧 |

### 統計カード（日ビュー）
4カラムグリッドで以下を表示:
- 投稿数（ソース別内訳をサブテキストで）
- 再生曲数（Last.fm）
- 歩数（Apple Watch）
- 天気（気温・天気概況・都市名）

### ソースフィルター
- 「すべて」ボタン: 全ソースをオンに戻す
- ソースタグ（ピル型トグルボタン）: クリックでオン/オフ
- オン時: ソース種別に応じた背景色・文字色
- オフ時: 無色・グレー文字
- 全オフ時: 「表示できる項目がありません」を表示

**ソースタグの色分け**
| 種別 | 背景 | 文字 | ボーダー |
|---|---|---|---|
| Misskey | #E1F5EE | #0F6E56 | #5DCAA5 |
| Mastodon | #EEEDFE | #3C3489 | #AFA9EC |
| Last.fm | #FCEBEB | #A32D2D | #F09595 |
| ヘルス | #FAEEDA | #854F0B | #EF9F27 |
| RSS | #EAF3DE | #3B6D11 | #97C459 |

### タイムライン
- ソースバッジ + 投稿内容 + 時刻 の3カラム構成
- バッジ: favicon（16x16px の `<img>`）+ コンパクトなサーバー名
  - 実装時: `https://<instance>/favicon.ico` を `<img>` で表示
  - フォールバック: ソース種別の絵文字（🍣 Misskey / 🐘 Mastodon / 🎵 Last.fm / 🍎 ヘルス / 🌐 RSS）
- ブースト/リノートは通常投稿と区別して表示
  - `BT: <元テキスト>` / `RN: <元テキスト>` のプレフィックス
  - 文字色: color-text-secondary（やや薄く）
- ホバー: 背景をcolor-background-secondaryに

**サーバー名の短縮ルール**
| フルドメイン | 表示名 |
|---|---|
| misskey.io | msk.io |
| tanoshii.site | tanoshii |
| sushi.ski | sushi.ski |
| mistodon.cloud | mistodon |
| mastodon.cloud | masto.cloud |
| msk.ilnk.info | ilnk |
| last.fm | last.fm |
| health（iPhone）| health |

## 技術スタック（ダッシュボード）

- バックエンド: Python / Flask
- テンプレート: Jinja2
- フロントエンド: Vanilla JS（フレームワークなし）
- CSS: CSS変数ベース（ライト/ダーク自動対応）
- グラフ: Chart.js（CDN）
- アクセス: Tailscale経由（http://<tailscale-ip>:5000）
